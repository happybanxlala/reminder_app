```## 5. 任務類型與時間規則

### 5.1 單次任務
- 使用者可建立只提醒一次的任務
- 單次任務不建立 `RecurringReminder`
- 單次任務至少需有日期
- 本階段 UI 若出現時間欄位，應明確標示目前仍以日期精度為主

### 5.2 固定時間
使用者語言：
- 每天
- 每週一
- 每月某日
- 每隔幾天

系統行為：
- `trackingMode = countdown`
- 必須有 `dueAt`
- 日期精度以日為準
- 若 `deferredDueAt != null`，則以 `deferredDueAt` 作為目前生效日期
- `triggerMode = 1` 時，當 `today >= effectiveDueAt - triggerOffsetDays` 進入提醒期
- `triggerMode = 2` 時，建立後即視為需提醒，`triggerOffsetDays` 無意義
- `triggerMode = 3` 時，僅在到期點或逾期後視為需提醒，`triggerOffsetDays = 0`

### 5.2.1 完成、跳過、取消與下一筆生成
- 單次任務完成後不再產生下一筆
- 固定任務完成後，若模板仍為 `pending` 且具 `repeatRule`，需自動產生下一筆
- 固定任務跳過後，若模板仍為 `pending` 且具 `repeatRule`，需自動產生下一筆
- 取消任務不會產生下一筆
- 若取消的任務屬於固定任務流程，系統可依產品規則將其模板轉為 `stopped`
- 重新啟用習慣模板時，系統應建立新的 pending 任務作為當前實例

### 5.3 從某天開始
使用者語言：
- 紀念日
- 第幾天
- 養寵物第幾天
- 戒某件事第幾天

系統行為：
- `trackingMode = accumulation`
- `triggerMode = 1`
- 當 `today >= effectiveDueAt - triggerOffsetDays` 進入提醒期
- `triggerOffsetDays = 0` 表示當天才可提醒
- UI 應可以 `startAt` 為起點顯示預覽文案，例如：`今天是第 N 天`

### 5.2.3 完成、跳過與下一筆生成
- 從某天開始任務加入了循環規則，完成或過後後，若模板仍為 `pending` 且具 `repeatRule`，需自動產生下一筆，`startAt`需直接繼承
- 系統可依產品規則將其模板轉為 `stopped`



## 8. UI / UX 需求

### 8.1 首頁資訊架構
首頁採 Top Tab，分三個分頁：
1. 今天
2. 接下來
3. 逾期

標題列右邊 button
4. 已完成（活動歷史）

標題列左邊button延伸的側欄
5. 定期任務
6. 從某天開始

### 8.2 今天
- 顯示今天需要優先處理的 pending 任務
- 固定時間任務主資訊顯示：`今天 / 明天 / 剩 X 天 / 逾期 X 天`
- 固定時間任務次資訊顯示：簡短規則摘要，如 `每天 / 每週`
- 固定時間任務可執行：完成、延後、跳過、取消
- 從某天開始任務主資訊顯示：`第 N 天`
- 從某天開始任務次資訊顯示：`距離重點日子剩 X 天`
- 從某天開始任務可執行：完成、跳過
- 為主要日常入口

### 8.3 接下來
- 顯示近期將到來的 pending 任務
- 作為預告與安排用途
- 與「今天」分開，避免資訊過載
- 卡片規則與「今天」一致，仍以主資訊/次資訊/補充資訊呈現

### 8.4 逾期
- 顯示近期已逾期的 pending 任務
- 用醒目的顏色強調
- 與「今天」分開，避免資訊過載
- 卡片規則與「今天」一致，仍以主資訊/次資訊/補充資訊呈現


### 8.5 已完成
- 顯示完成與跳過的任務歷史
- 依 `updatedAt` 由新到舊排序
- 主要用於回顧，不作為編輯入口

### 8.6 定期任務
- 作為定期任務模板管理頁，而非每日工作頁
- 顯示：
  - 標題
  - 「定期任務」標籤
  - 簡短規則摘要
  - 狀態
- 操作：
  - `pending`：可編輯 / 暫停 / 取消
  - `stopped`：可編輯 / 啟用
  - `canceled`：唯讀

補充：
- 視覺上應避免表格式堆疊
- 卡片應更接近日常產品，而非 admin panel

### 8.6 新增定期任務流程（Wizard）
新增流程應採 step-based wizard，而非一次顯示所有技術欄位。

#### Step 1：輸入內容
- 你想提醒什麼？（必填）
- 分類（選填）
- 備註（選填）

#### Step 2：是否需要重複
選項：
- 不需要：只提醒一次
- 需要：系統會自動幫你安排下一次

#### Step 3A：單次任務設定
- 目標完成日期（必填）

#### Step 3B：固定時間設定
- 第一次完成日期（必填）
- 頻率
- 日期基準
- 設定顯示任務規則摘要：
  - 提醒期時顯示（提前多少天提醒？）／輸入天數
  - 立即提醒
  - 當天提醒

### 8.7 新增從某天開始流程（Wizard）
- 作為從某天開始任務模板管理頁，而非每日工作頁
- 顯示：
  - 標題
  - 「從某天開始」標籤
  - 簡短規則摘要
  - 狀態
- 操作：
  - `pending`：可編輯 / 暫停
  - `stopped`：唯讀

#### Step 1: 輸入內容
- 你想紀錄什麼？（必填）
- 分類（選填）
- 起始日期（必填）
- 備註（選填）
- 顯示方式：第幾天 / 第幾週 / 第幾年

#### Step 2: 重點日子
- 加入循環基準（選填），例如：`每 100 天`、`每 1 個月`
- 加入指定日期（選填），不限次數，題示在頁面
- 重點日子的提醒期時顯示（提前多少天提醒？）／輸入天數
- 如未加入循環及指定日期，可略過輸入

### 8.8 UX 約束
- UI 不應讓使用者先理解工程分類，再開始建立任務
- 類型選擇應延後到「已確認需要重複」之後再出現
- 設定頁不作為類型建立入口
- UI 不得直接出現「起計式」「倒數式」「trackingMode」「triggerMode」
```

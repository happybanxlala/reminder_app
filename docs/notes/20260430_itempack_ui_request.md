1. [UI]整合「itemsManagement」和「itemsManagement」兩個功能頁為一頁:
    內容:
    - 新增事項(item)按鈕
    - 能收合/展開的責任包(item pack)列表

2. 新增事項(item)功能:
    - 點擊「新增事項」按鈕，彈出表單讓使用者輸入事項名稱、設定事項類型（固定節奏、彈性處理、庫存）以及是否加入責任包。
    - 使用者若選擇加入責任包，則顯示目前的責任包列表供使用者選擇，並且可以選擇「不加入任何責任包」，代表加入Default item pack。
    - 新責任包的按鈕則在責任包列表欄位旁，點擊後彈出表單讓使用者輸入責任包名稱，創建新的責任包。
    - 使用者填寫完畢後，點擊「確認」按鈕，將新事項及責任包加入資料庫並顯示在列表中.
    - 使用者取消新增事項，點擊「取消」按鈕，即使責任包已經創建也不會將新事項加入資料庫，並且不會顯示在列表中。

3. 能收合/展開的責任包(item pack)列表
    - 在事項列表中，責任包(item pack)以可收合/展開的方式顯示。
    - 預設情況下，所有責任包都是收合狀態，使用者可以點擊責任包名稱來展開或收合該責任包下的事項。
    - 展開狀態下，顯示該責任包下的所有事項；收合狀態下，隱藏該責任包下的事項。
    - 使用者可以透過這種方式更方便地管理和瀏覽大量事項，尤其是當有多個責任包和許多事項時.

第二輪更新

強調字眼定義：
- 尚未開始：當天在開始日期(AnchorDate)以前
- 進入warning期間：參考 warningAfter/warningBefore
- 進入danger期間：參考 在dangerAfter/dangerBefore

針對責任包下的事項顯示調整：
- 事項卡只會顯示事項名稱、類型和到期日期。其他資訊將在點擊事項卡後的詳細頁面/長按中顯示（顯示方式是彈窗形式）
- 類型顯示將使用圖示來區分（例如：固定節奏、彈性處理、庫存分別使用不同的圖示），並置於事項名稱旁邊。
- 使用編輯圖示（通常是鉛筆圖示）來表示編輯操作，並且這些操作選項將放在事項卡的右側。
- 使用overflow menu（通常是三個點的圖示）來顯示完成、跳過、查看歷史紀錄等操作選項，並且這些選項將放在事項卡的右側，與編輯圖示並列。

1. 針對不同類型顯示圖示的：
- 固定節奏：使用一個時鐘圖示，代表需要在固定的時間點處理。
- 彈性處理：使用一個彈性的圖示，例如一個彈力繩，代表可以根據情況彈性處理。
- 庫存：使用一個箱子圖示，代表需要根據庫存狀況處理。
2. 針對不同類型的狀態顯示：
狀態顯示將改為在事項名稱下方以小字體顯示、並以顏色代表狀態區分，具體如下：
- 固定節奏：顯示下一個到期日期，例如「下次截止：2024/07/01」﹔進入「warning」期間時，顯示「要留意：距截止還有3天」並且文字顏色變為橙色﹔進入「danger」期間時，顯示「需要處理：距截止還有1天」並且文字顏色變為紅色。未開始的事項則顯示「尚未開始」並且文字顏色為灰色。
- 彈性處理：顯示距上次處理的時間，例如「距上次處理：2天前」﹔進入「warning」期間時，顯示「要留意：距上次處理已經有5天了」並且文字顏色變為橙色﹔進入「danger」期間時，顯示「需要處理：距上次處理已經有7天了」並且文字顏色變為紅色。未開始的事項則顯示「尚未開始」並且文字顏色為灰色。
- 庫存：顯示目前庫存數量，例如「剩餘5天庫存量」﹔進入「warning」期間時，顯示「要留意：庫存剩3天了」並且文字顏色變為橙色﹔進入「danger」期間時，顯示「需要處理：只剩1天庫存量」並且文字顏色變為紅色。未開始的事項則顯示「尚未開始」並且文字顏色為灰色。
3. 事項卡點擊後的詳細頁面/長按顯示的彈窗內容：
- 彈窗將顯示事項的詳細資訊，包括事項名稱、類型、開始日期、上次處理日期、庫存數量、是否加入責任包、以及其他相關資訊。
- 只提供唯讀資訊，不提供在彈窗中編輯的功能。編輯功能將放在事項卡的overflow menu中。
- 彈窗將有一個「關閉」按鈕，讓使用者可以關閉彈窗返回事項列表。
4. overflow menu的操作選項，分為3個區塊並以分隔線區分：
- 第一區塊：包含「完成」和「跳過」兩個選項，讓使用者可以標記事項的完成狀態或選擇跳過該事項。事項尚未開始時，
- 第二區塊：包含「詳細資訊」和「查看歷史紀錄」兩個選項，讓使用者可以查看事項的詳細資訊彈窗或查看該事項的歷史紀錄。
- 第三區塊：包含「暫停」和「封存」兩個選項，並以紅字標示，讓使用者可以暫時停止該事項的提醒或將該事項封存起來。
- 事項尚未開始時，第一區塊的「完成」和「跳過」選項將會是灰色不可點擊的狀態，以避免使用者對尚未開始的事項進行不適當的操作。
- 使用者選擇「完成」後，事項將被標記為已完成，並且在列表中顯示為已完成的狀態（例如：文字變灰並加上刪除線）。使用者選擇「跳過」後，事項將被標記為已跳過，並且在列表中顯示為已跳過的狀態（例如：文字變灰並加上斜線）。
- 若事項類型是彈性處理，同時並不在warning或danger期間，使用者選擇「完成」後，將會有一個確認彈窗提示使用者在不預期時間下完成事項，詢問使用者是否確定要標記事項為已完成，以避免誤操作。其他類型的事項則不受此限制，使用者選擇「完成」後將直接標記事項為已完成。「跳過」則不受此限制，使用者選擇「跳過」後將直接標記事項為已跳過。
- 頁面刷新後，事項卡將會顯示下一輪的狀態更新（例如：如果是固定節奏事項，則會顯示下一個到期日期；如果是彈性處理事項，則會顯示距上次處理的時間；如果是庫存事項，則會顯示更新後的庫存數量）。
- 使用者選擇「詳細資訊」後，將會顯示事項的詳細資訊彈窗（同事項卡點擊後的彈窗內容）。
- 使用者選擇「查看歷史紀錄」後，將會跳轉到該事項的歷史紀錄頁面，顯示該事項的過去處理紀錄和狀態變化歷史。
- 使用者選擇「暫停」或「封存」後，將會有一個確認彈窗詢問使用者是否確定要執行該操作，以避免誤操作.
- 若使用者確認暫停，該事項將會暫時停止提醒，並且在列表中顯示為暫停的狀態（例如：卡片變灰並加上暫停圖示）。若使用者確認封存，並且在列表中不再顯示。
